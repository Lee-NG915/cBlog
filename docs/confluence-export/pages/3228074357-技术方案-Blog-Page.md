## 背景

我们此前已将 Blog 相关内容从 Knight 迁移至 Storyblok。然而，在 US 站点，Blog 的访问量增长超出预期，导致 **knight-us space** 的 API 请求超出原定配额。因此，我们计划将 Blog 相关内容从 **OnePiece** 迁移至 **JoyBoy**，主要有两大优势：

- **降低 Storyblok 请求量**：借助 Next.js 的接口缓存机制，减少对 Storyblok API 的依赖，提高访问效率。
- **提升页面性能**：复用 JoyBoy 在 PLA 方面的优秀实践，为 Blog 提供更高效的页面渲染，优化用户体验。
## 迁移内容

### Blog List Page

path: /blog

#### 大致变动

Blog List 的内容获取方式有所调整：

- **OnePiece**：通过 Node 获取所有 Blog List 目录，并在前端采用假分页渲染，以减少 API 请求次数。
- **JoyBoy**：直接在页面内获取 Blog 数据，依托接口缓存，无需担心过多 API 请求。同时，解决了 OnePiece 方案中 **数据量过大** 和 **轮询翻页** 的问题，进一步优化页面性能。
此外，在请求接口时，我们基于最新的 **Storyblok package** 版本，添加了 **starts_with** 参数，有效隔离非当前市场文件夹内的数据，避免了额外的过滤处理，提高了数据获取的效率。

### Blog Page

path: /blog/:slug

#### 大致变动

通过让 **Menu** 和 **Footer** 自主管理数据，不仅减少了页面体积，还提升了加载效率。此外，我们还增加了 **10 分钟的页面缓存和接口缓存**，进一步优化访问性能。

### Storyblok Components（Blog 使用的）

弃用原先手动发送 tracking 的方式，改为与 **JoyBoy PLA** 一致的 tracking 方案。通过标记组件外层标签，让 **GTM** 自动获取数据，实现更高效、准确的埋点追踪。

然后拆分。
首先，通过使用集成的 getStoryWithCache 方法，获取到 story 的数据。

```
async getStoryWithCache<T = any>(
  slug: string,
  params?: Partial<ISbStoryParams>,
  options?: {
    tags?: string[];
    revalidate?: number;
  }
) {
  const cacheKey = `sb-${slug}-${JSON.stringify(params)}`;
  const sbSlugTag = `sb-${slug}`;
  return cache(() => this.getStory<T>(slug, params), [cacheKey], {
    tags: ['storyblok', sbSlugTag, ...(options?.tags || [])],
    revalidate: options?.revalidate || EcEnv.NEXT_PUBLIC_STORYBLOK_REVALIDATE_TIME,
  })();
}
```

然后，获取需要使用到的 Product 以及 相关的 SEO 信息。

```
const storyblokProductList = {};
const productIds = new Set();

for (const item of story.content?.body || []) {
  if (item.component === 'blog-banner') {
    if (item.image?.[0]) {
      blogBannerInfo.image = item.image?.[0].desktop_url || '';
    }
    blogBannerInfo.author = item?.author;
    blogBannerInfo.title = item?.articleTitle || '';
    item.publishDate = story?.first_published_at || '';
  }

  if (item.component === 'detailed-product-listing' || item.component === 'simple-product-listing') {
    if (item?.product_id) {
      productIds.add(item.product_id);
    }
  }

  if (item.component === 'detailed-listing' || item.component === 'simple-listing') {
    if (item.items?.length > 0) {
      item.items
        .filter(
          (subItem) =>
            subItem.component === 'detailed-product-listing' || subItem.component === 'simple-product-listing'
        )
        .forEach((product) => {
          if (product.product_id) {
            productIds.add(product.product_id);
          }
        });
    }
  }
}
```

Storyblok Components 的改动体现在使用给标签增加 attribute 来自动完成 tracking 流程：

```
export function DtStack({
  children,
  uid,
  componentName,
  elementName,
  useImpression = false,
  ...restProps
}: DtStackProps) {
  const pathname = usePathname();

  const impressionProps = useImpression ? addImpressionFlag(uid) : {};
  const trackingProps = createDataTrackingData({
    pathname,
    module: componentName,
    elementName: elementName,
  });

  return (
    <Stack {...impressionProps} {...trackingProps} {...restProps}>
      {children}
    </Stack>
  );
}
```
