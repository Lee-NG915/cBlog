## i18n 最终呈现

> ### 基于 URL 路径名的语言环境通信

为了在浏览器与 Web 服务器之间建立高效的语言环境通信，可基于 URL 路径名（如 `/en`、`/zh`）来识别用户的语言偏好。同时，Web 服务器可在响应时附加 locale Cookie，以优化浏览器的语言自动检测功能。

**URL 语言路径识别**

- 访问 `/zh/page/...` 时，服务器返回简体中文内容。
- 访问 `/page/...` 时，服务器返回英文内容。`/en`作为默认语言路径，不显式呈现在路径中。
- **浏览器 **`Accept-Language`** 头解析**

- 浏览器请求时，携带 `Accept-Language: en` 以表明语言偏好。前端与服务端通信时，可基于`Accept-Language`属性，保持请求和响应的语言版本一致性。
- **语言环境 Cookie 设定**

- Web服务器可在响应时设置 Cookie（如 `prefered_language = en`），便于后续请求识别用户语言偏好。

**查看下方时序图来理解具体实现方案：**

```
sequenceDiagram
    User->>+Browser: 
    loop Default Locale:en 
    Browser->>+Web Server: https://www.castlery.com/sg 
    Web Server->>+Web Server:[middleware] Rewrite url & Set response cookies
    Note left of Web Server: Response Cookie:preferred_language:en
    Web Server->>+Browser: https://www.castlery.com/sg(/en 隐式存在于URL上) 
    Note right of Browser: Request Headers: Accept-Language:en
    Browser->>+Backend Server: https://www.castlery.com/sg/api/xxxx
    Backend Server->>+Browser: Response content in en format
    end
    loop Custom Locale:zh 
    Browser->>+Web Server: https://www.castlery.com/sg/zh
    Web Server->>+Web Server: [middleware] Set response cookies
    Note left of Web Server: Response Cookie:preferred_language:zh
    Web Server->>+Browser: https://www.castlery.com/sg/zh (/zh 显式存在于URL上) 
     Note right of Browser: Request Headers: Accept-Language:zh
    Browser->>+Backend Server: https://www.castlery.com/sg/api/xxxx
    Backend Server->>+Browser: Response content in zh format
    end
```

 在上图中可以看到，Web Server接收到访问请求，会进行URL解析，适当的重写URL（基于nextjs rewrite res实现），并在Response Cookies携带cookie `prefered_language` ,浏览器存储这个cookie。如果前端开启了localeDetection，则cookie `prefered_language`会覆盖当前路由中的locale。

 与后端通信时，在请求头Request Headers携带`Accept-Language:en;` 来保证前端语言版本一致。

**3.2 i18nRouterHandler**

通过用户场景解析，来理解i18nRouterHandler如何工作：

**场景一：用户初次访问网站，未设置语言偏好，则使用默认语言，重写响应URL**

![](attachment)

**场景二：用户访问过网站，曾设置语言偏好是zh，则重定向响应URL**

![](attachment)

**场景三：用户访问网站，手动选择语言偏好为en, 则正常返回(****path上的locale会覆盖cookie中的语言****)**

```
// 兼容i18n路由，实现对URL重写或者重定向
export const i18nRouterHandler = (request: NextRequest) => {
  if (request.nextUrl.pathname === '/') {
    return NextResponse.next();
  }
  //以 /_next 开头的路由都是由框架自动生成并用于内部处理的。这些路由通常用于加载静态资源、脚本和其他与应用程序相关的静态文件。
  if (
    !supportRegions.some((region) => request.nextUrl.pathname.startsWith(`/${region}`)) &&
    !request.nextUrl.pathname.startsWith('/_next')
  ) {
    return NextResponse.redirect(new URL(`/`, request.url));
  }

  const regionInPathname = supportRegions.find((region) => request.nextUrl.pathname.startsWith(`/${region}`));
  if (regionInPathname) {
    const pathnameWithoutRegion = request.nextUrl.pathname.replace(`/${regionInPathname}`, '');
    // 如果pathname中包含语言，则重置i18ncookie，再继续response
    if (languages.some((lang) => pathnameWithoutRegion.startsWith(`/${lang}`))) {
      const localeInPathname = languages.find((lang) => pathnameWithoutRegion.startsWith(`/${lang}`));
      const response = NextResponse.next();
      localeInPathname &&
        makePersistenceHandles().preferredLanguage.setItem(localeInPathname, { req: request, res: response });
      makePersistenceHandles().preferredRegion.setItem(regionInPathname, { req: request, res: response });
      return response;
    }

    const locale = request.cookies.get(preferredLanguage)?.value ?? '';
    // 如果pathname&cookie中都不包含语言，则重定向到默认语言
    if (!locale) {
      const response = NextResponse.rewrite(
        new URL(`/${regionInPathname}/${fallbackLng}${pathnameWithoutRegion}`, request.url)
      );
      makePersistenceHandles().preferredLanguage.setItem(fallbackLng, { req: request, res: response });
      makePersistenceHandles().preferredRegion.setItem(regionInPathname, { req: request, res: response });
      return response;
    }

    // 如果pathname中不包含语言,cookie中包含合法语言，且是默认语言，则重写URL
    if (isSupportedLanguage(locale) && locale === fallbackLng) {
      const newPathname = `/${regionInPathname}/${locale}${pathnameWithoutRegion}`;
      const response = NextResponse.rewrite(new URL(newPathname, request.url));
      makePersistenceHandles().preferredLanguage.setItem(locale, { req: request, res: response });
      makePersistenceHandles().preferredRegion.setItem(regionInPathname, { req: request, res: response });
      return response;
    }

    // 如果pathname中不包含语言,cookie中包含合法语言，且不是默认语言，则重定向到带语言的URL
    const response = NextResponse.redirect(
      new URL(`/${regionInPathname}/${locale}${pathnameWithoutRegion}`, request.url)
    );
    makePersistenceHandles().preferredLanguage.setItem(locale, { req: request, res: response });
    makePersistenceHandles().preferredRegion.setItem(regionInPathname, { req: request, res: response });
    return response;
  }
  return NextResponse.redirect(new URL(`/`, request.url));
};
```

### 4. 开发i18n client hook & server hook

**client hook ：**

- `useTranslation`
- `monorepo-i18n/src/lib/scripts/useTranslation.client.ts`
```
'use client';

import { useEffect, useState } from 'react';
import { useTranslation as useTranslationOrg } from 'react-i18next';
import { makePersistenceHandles } from '@castlery/shared-persistence-kit';
import { initI18n } from './init';
import { LocalesNamespace } from '../types';

const runsOnServerSide = typeof window === 'undefined';
const persistenceHandles = makePersistenceHandles(); // 避免每次渲染都创建新实例

initI18n({}); //初始化
export function useTranslation(lng: string, ns: LocalesNamespace, options?: { keyPrefix?: string }) {
  const { i18n, ...rest } = useTranslationOrg(ns, options);
  const [ready, setReady] = useState(() => i18n.hasResourceBundle(lng, ns));
  const [activeLng, setActiveLng] = useState(i18n.resolvedLanguage);

  useEffect(() => {
    if (ready) return;
    i18n.loadNamespaces(ns, () => {
      setReady(true);
    });
  }, [ready, ns, i18n]);

  useEffect(() => {
    if (activeLng === i18n.resolvedLanguage) return;
    setActiveLng(i18n.resolvedLanguage);
  }, [i18n.resolvedLanguage]); // 移除 `activeLng` 依赖，防止不必要的 `setState`

  useEffect(() => {
    if (!lng || i18n.resolvedLanguage === lng) return;
    i18n.changeLanguage(lng);
  }, [lng, i18n]);

  useEffect(() => {
    if (persistenceHandles.preferredLanguage.getItem() === lng) return;
    persistenceHandles.preferredLanguage.setItem(lng);
  }, [lng]);

  if (runsOnServerSide && lng && i18n.resolvedLanguage !== lng) {
    i18n.changeLanguage(lng, (err) => {
      if (err) console.error('i18next change language error:', err);
      persistenceHandles.preferredLanguage.setItem(lng);
    });
  }

  return { ...rest, i18n, ready };
}
```

## **在CSR和SSR中初始化i18n的区别**

 如何在服务端渲染SSR的页面中，初始化并使用i18n，详见下文：

- SSR场景下i18n的初始化是异步阻塞的，在i18n初始化完成之后，才返回翻译函数*t*。
```
const { t } = await useTranslationServer(Languages.EN, LocalesNamespace.WEB);
```

- 
 如何在client页面初始化并使用i18n? 

- CSR场景下，i18n初始化非阻塞，在引入`useTranslation`时，即执行初始化。
- 返回 ready字段，来反应翻译函数*t*是否可用。

```
const { t, ready } = useTranslation(Languages.EN, LocalesNamespace.WEB)
```
### 如何兼容SSR？

在NextJs 中实现 internationalization，必须考虑SSR场景。

我们概述一下SSR的渲染流程：

> SSR场景下，i18n的初始化和数据加载都在第   **3  **步实现：

在服务器上执行页面渲染前，在服务端执行`getServerSideProps`或者`getInitialProps`，用于获取页面所需的初始数据，包括 `i18n` 翻译数据。

NextJs 13/14则不需要使用`getServerSideProps`等hook，直接在page中获取。

### 在SSR中使用i18n需要注意什么？

1. - ssr中每个页面需要独立初始化i18n实例，避免并行请求时翻译数据的混乱。也就是在page level 做 i18n.init。
2. - 可以使用`serverSideTranslations` 加载翻译数据（`locales/xxx/transtlation.json`）。
- `serverSideTranslations` 用于在服务器端加载翻译文件，并将它们作为页面组件的 props 传递。
- `useTranslation` 钩子用于在组件中访问翻译函数 `t`。

 每增加一个locales文件，都应该在Resource中暴露出来。

```
import { LocalesNamespace } from './namescape';
// import all namespaces (for the default language, only)
import CartTranslation from '../locales/en/modules/cart/sg.json';
import WebTranslation from '../locales/en/web/sg.json';
import PosTranslation from '../locales/en/pos/sg.json';
import SharedTranslation from '../locales/en/shared/sg.json';

export type CartResource = typeof CartTranslation;
export type WebResource = typeof WebTranslation;
export type PosResource = typeof PosTranslation;
export type SharedResource = typeof SharedTranslation;

export type Resource = {
  [LocalesNamespace.MODULES_CART]: CartResource;
  [LocalesNamespace.SHARED]: SharedResource;
  [LocalesNamespace.WEB]: WebResource;
  [LocalesNamespace.POS]: PosResource;
};
```
