# AWS WAF v2 迁移技术方案设计文档

## 1. 需求背景

### 1.1 项目背景

当前系统使用 AWS WAF v1 与 ALB 集成提供 Web 应用防护。由于 AWS WAF v1 即将被淘汰，同时存在异常流量影响数据采集质量的问题，需要通过迁移到 WAF v2 来增强安全防护能力，并通过其高级特性更好地管控异常流量。

### 1.2 现状与问题

```
flowchart TD
    subgraph Current[当前WAF v1限制]
        A1[规则限制多] --> B1[无法满足复杂防护]
        A2[配置分散] --> B2[管理维护困难]
        A3[功能受限] --> B3[新特性无法使用]
        A4[异常流量] --> B4[影响数据采集]
    end

    subgraph Future[WAF v2优势]
        C1[规则更灵活] --> D1[精细化防护]
        C2[中心化配置] --> D2[简化管理维护]
        C3[新增功能] --> D3[增强防护能力]
        C4[流量管控] --> D4[减少异常干扰]
    end

    Current --> Future
```

### 1.3 迁移目标与收益

## 1.3.1 核心目标

1. - **技术升级目标**

- 完成从 WAF v1 到 v2 的平滑迁移
- 利用 WAF v2 的新特性增强防护能力
- 实现配置的中心化管理
2. - **业务保护目标**

- 识别和过滤异常流量
- 保护数据采集端点
- 确保搜索引擎正常访问
- 防止资源滥用
## 1.3.2 预期收益

1. - **技术收益**

- 使用 AWS 托管规则降低维护成本
- 通过规则组简化管理
- 利用新特性提升防护精度
2. - **防护收益**

- 识别和控制异常访问行为
- 保护关键业务接口
- 降低资源滥用风险
- 确保正常业务访问不受影响

### 4.2 实施步骤

#### 4.2.1 准备阶段

```
# 1. 导出现有WAF规则
aws waf-regional get-web-acl --web-acl-id ${ACL_ID} --region ${REGION}

# 2. 创建新的Web ACL
aws wafv2 create-web-acl \
    --name "Production-WebACL" \
    --scope REGIONAL \
    --default-action Allow={} \
    --description "Production environment WAF rules" \
    --region ${REGION}

# 3. 配置日志
aws wafv2 put-logging-configuration \
    --logging-configuration \
    ResourceArn=${WEBACL_ARN},LogDestinationConfigs=[${KINESIS_FIREHOSE_ARN}]
```

#### 4.2.2 规则迁移

```
迁移步骤:
  1. 基础规则迁移:
     - 导出现有规则
     - 转换规则格式
     - 部署新规则
  
  2. 数据质量规则配置:
     - GA相关规则
     - GTM相关规则
     - SEO优化规则
  
  3. 验证与优化:
     - 规则测试
     - 效果验证
     - 性能优化
```

#### 4.2.3 切换步骤

```
# 1. 解除WAF v1关联
aws waf-regional disassociate-web-acl \
    --resource-arn ${ALB_ARN} \
    --region ${REGION}

# 2. 关联WAF v2
aws wafv2 associate-web-acl \
    --web-acl-arn ${WEB_ACL_ARN} \
    --resource-arn ${ALB_ARN} \
    --region ${REGION}
```

### A. AWS WAF v2 功能列表

| 功能类型 | 说明 | v1 vs v2 |
| 规则能力 | 支持更复杂的规则逻辑 | v2 增强 |
| 托管规则 | AWS 维护的规则集 | v2 更多 |
| 地理位置 | 基于位置的访问控制 | v2 新增 |
| 速率限制 | 基于请求率的控制 | 两者都有 |
| 自定义响应 | 自定义拦截响应 | v2 新增 |
| 日志能力 | 详细的访问日志 | v2 增强 |


### B. 常用命令参考

```
# 查看 Web ACL

aws wafv2 list-ip-sets --scope REGIONAL --region ${REGION}

aws wafv2 get-logging-configuration \

    --region ${REGION}

aws cloudwatch get-metric-statistics \
    --namespace AWS/WAFV2 \
    --metric-name BlockedRequests \
    --dimensions Name=WebACL,Value=${WEB_ACL_NAME} \
    --start-time ${START_TIME} \
    --end-time ${END_TIME} \
    --period 300 \
    --statistics Sum
```
