# **CloudFront Functions + KVS 统一开关（生产版本）**

## **目标**

- **同一个 CloudFront distribution** 同时承载 .com 与 .co
- 支持：

- **按 region 的维护模式跳转**（/{region}/maintenance）
- **域名跳转**（.com ↔ .co），并且能：

- **今天 Phase 1：302**
- **明天 Phase 2：301**
- 随时 off 一键关闭
- 两个 Function（有维护/无维护）**共享同一套 KVS key**，只改一次 KVS 就能控制。
---

# **1. KVS Key 规范（统一约定）**

## **1.1 维护模式（按 region）**

> 你已有，无需改。

- us_maintenance = true|false
- sg_maintenance = true|false
- au_maintenance = true|false
- uk_maintenance = true|false
- ca_maintenance = true|false
说明：

- true：开启维护，进入维护页
- 其他值/缺失：视为关闭
## **1.2 域名跳转（全局）**

新增两个全局 key：

### **A) 域名跳转模式**

- domain_redirect_mode = off | com_to_co | co_to_com
含义：

- off：不跳转
- com_to_co：*.castlery.com -> *.castlery.co
- co_to_com：*.castlery.co -> *.castlery.com
### **B) 域名跳转状态码（可选但推荐）**

- domain_redirect_status = 301 | 302
说明：

- 如果该 key 存在且值为 301/302，则**强制覆盖**跳转的状态码
- 如果不设置：

- com_to_co 默认 302
- co_to_com 默认 301
> 你这次的需求（今天 302，明天 301，且方向仍 com->co）推荐用：

- domain_redirect_mode = com_to_co
- 今天：domain_redirect_status = 302
- 明天：domain_redirect_status = 301

# **2. CloudFront Function（最终代码）**

## **2.1 Function A：Maintenance + Domain Redirect（推荐默认挂这份）**

适用：你想要维护模式和域名跳转都在一个函数内完成。

```
import cf from 'cloudfront';

const VALID_REGIONS = ['us', 'sg', 'au', 'uk', 'ca'];

function getRegion(uri) {
  const regionString = uri.split('/')[1];
  const normalized = regionString ? regionString.toLowerCase() : '';
  return VALID_REGIONS.includes(normalized) ? normalized : '';
}

function buildQueryString(querystring) {
  const keys = Object.keys(querystring || {});
  if (keys.length === 0) return '';

  const parts = [];
  for (let i = 0; i < keys.length; i++) {
    const k = keys[i];
    const item = querystring[k];
    const ek = encodeURIComponent(k);

    if (item.multiValue && item.multiValue.length > 0) {
      for (let j = 0; j < item.multiValue.length; j++) {
        parts.push(ek + '=' + encodeURIComponent(item.multiValue[j].value));
      }
    } else if (item.value !== undefined) {
      parts.push(ek + '=' + encodeURIComponent(item.value));
    }
  }
  return parts.length ? ('?' + parts.join('&')) : '';
}

function isMaintenancePath(uri) {
  return uri === '/maintenance' || uri.endsWith('/maintenance');
}

function mapHost(host, mode) {
  if (mode === 'com_to_co') {
    if (host === 'castlery.com') return 'castlery.co';
    if (host.endsWith('.castlery.com')) return host.replace(/\.castlery\.com$/, '.castlery.co');
    return '';
  }

  if (mode === 'co_to_com') {
    if (host === 'castlery.co') return 'castlery.com';
    if (host.endsWith('.castlery.co')) return host.replace(/\.castlery\.co$/, '.castlery.com');
    return '';
  }

  return '';
}

async function handler(event) {
  const request = event.request;
  const kvs = cf.kvs();

  const uri = request.uri || '/';
  const host = (request.headers.host && request.headers.host.value)
    ? request.headers.host.value.toLowerCase()
    : '';
  const region = getRegion(uri);

  // -------------------------
  // 1) Maintenance redirect (per-region)
  // -------------------------
  if (!isMaintenancePath(uri)) {
    try {
      let isInMaintenanceMode = false;

      if (region) {
        const kvValue = await kvs.get(`${region}_maintenance`);
        isInMaintenanceMode = (kvValue === 'true' || kvValue === true);
      }

      if (isInMaintenanceMode) {
        const maintenanceUrl = region ? `/${region}/maintenance` : `/maintenance`;

        return {
          statusCode: 302,
          statusDescription: 'Found',
          headers: {
            location: { value: maintenanceUrl },
            'cache-control': { value: 'no-store, no-cache, must-revalidate' },
          },
        };
      }
    } catch (err) {
      console.log('Error reading maintenance from KVS: ' + (err && err.message ? err.message : String(err)));
    }
  }

  // -------------------------
  // 2) Domain redirect (global)
  // -------------------------
  // 建议：maintenance 页面本身不做域名跳转，避免排障被带走
  if (isMaintenancePath(uri)) return request;

  try {
    const rawMode = await kvs.get('domain_redirect_mode');
    const mode = (rawMode || '').toString().trim().toLowerCase() || 'off';

    if (mode === 'off') return request;

    const targetHost = mapHost(host, mode);
    if (!targetHost) return request;

    // default status
    let statusCode = (mode === 'co_to_com') ? 301 : 302;

    // optional override
    try {
      const rawStatus = await kvs.get('domain_redirect_status');
      const s = (rawStatus || '').toString().trim();
      if (s === '301' || s === '302') statusCode = parseInt(s, 10);
    } catch (err) {
      // ignore
    }

    const qs = buildQueryString(request.querystring);
    const location = 'https://' + targetHost + uri + qs;

    return {
      statusCode,
      statusDescription: statusCode === 301 ? 'Moved Permanently' : 'Found',
      headers: {
        location: { value: location },
        'cache-control': { value: 'no-store, no-cache, must-revalidate' },
      },
    };
  } catch (err) {
    console.log('Error reading domain redirect from KVS: ' + (err && err.message ? err.message : String(err)));
  }

  return request;
}
```

# **3. Behavior 绑定建议**

- 原本走OP的走函数B
- 走joyboy的走函数A
注意：

- 由于 .com 和 .co 都进同一个 distribution，**必须依赖 Host 判断**（函数里已处理）。
- 不要把跳转函数绑到 Default (*)，否则全站都会跳。

# **4. Phase 切换步骤（按你的需求）**

## **Phase 1（今天）：**

## **.com -> .co**

## **302**

在 KVS 设置：

- domain_redirect_mode = com_to_co
- domain_redirect_status = 302（推荐显式写上）
效果：

- [https://*.castlery.com/us/cart…](#) 会 302 到 [https://*.castlery.co/us/cart…](#)
- [https://*.castlery.co/us/cart…](#) 不跳（因为只对 .com 生效）
## **Phase 2（明天）：同方向**

## **.com -> .co**

## **改 301**

只改 KVS：

- domain_redirect_mode = com_to_co（不变）
- domain_redirect_status = 301
效果：

- .com -> .co 变成 301
## **关闭（随时回滚）**

- domain_redirect_mode = off
- （可选）删除 domain_redirect_status

> 用 -I 看响应头，确认 Location、状态码和 query 是否保留。

### **验证**

### **.com**

### **会跳**

```
curl -I 'https://www.castlery.com/us/cart?coupon=abc&x=1'
```

你应该看到：

- HTTP/2 302（或明天 301）
- Location:
### **验证**

### **.co**

### **不跳（同方向 com->co）**

```
curl -I 'https://www.castlery.co/us/cart?coupon=abc&x=1'
```

你应该看到：

- HTTP/2 200/304（或正常站内响应）
- **不应出现** Location

# **6. 常见风险与规避**

1. - **重定向循环**
- 只有当你同时开启 com_to_co 和 co_to_com 才会 loop。
- 所以 KVS **mode 永远只允许一个方向**，不要同时开。
1. - **缓存导致明天切 301 不生效**
- 代码里已经加了 cache-control: no-store…
- 明天切完可以再用 curl 验证状态码是否正确。
1. - **query 参数异常**
- 代码已做 encodeURIComponent，避免出现 &、空格、中文导致的错乱。
