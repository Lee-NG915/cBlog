# 1 背景

### 1.1 业务背景

跨区域电商网站在多市场运营时，功能特性的合理划分与管理至关重要。为了确保不同市场的需求得到精准满足，同时保持系统的灵活性与可维护性，我们建议将**功能特性管理划分为以下三个层级**：

1. - **业务功能特性（领域业务模型）**

- 作为领域模型的一部分，这类功能特性承载核心业务逻辑，具有较高的稳定性和一致性。
- 变更通常伴随着领域模型的调整，需经过严格的版本管理和代码演进策略。
2. - **市场差异化功能特性（通常为业务 Hardcode）**

- 适用于不同市场的专属业务逻辑，如特定的支付方式、物流规则、税务计算等。
- 由于其高度市场相关性，应通过合理的架构设计（如策略模式、配置驱动开发等）减少代码硬编码，提升可维护性。
3. - **动态功能开关（Feature Flag）**

- 主要用于 A/B 测试、新功能灰度发布、阶段性功能切换等场景。
- 通过配置管理或开关平台控制，能够灵活启用或禁用功能，确保功能调整的高效性与安全性。
| **功能特性** | **适用场景** | **变更频率** | **示例** |
| **业务功能特性** | 核心业务逻辑 | 低 | 订单计算、库存管理 |
| **市场功能特性** | 不同市场业务规则 | 中等 | 税率、支付、物流规则 |
| **动态功能开关Feature Flag** | 实验功能 / A/B 测试/第三方服务 | 高 | 新 UI、促销开关 |


### 1.2 技术背景

**Joyboy** 采用 **Monorepo**（单一代码库）架构，并遵循 **DDD（领域驱动设计）** 原则，以确保多个市场的业务需求能够高效组织和管理。

- Monorepo 是一种代码管理策略，即将多个子项目（如前端、后端、微服务等）存放在同一个代码仓库中，而不是分散在多个独立的仓库中。这种方式能够提高代码复用性、统一依赖管理，并提升团队协作效率。例如，在 Joyboy 项目中，多个市场的业务逻辑可以共享核心代码，同时通过模块化设计实现灵活扩展。
- DDD（Domain-Driven Design，领域驱动设计）是一种软件架构设计方法，旨在通过明确领域模型，聚焦业务逻辑，以提高系统的可维护性和扩展性。在 Joyboy 项目中，DDD 主要体现在：

- **领域分层**：将系统拆分为领域层、应用层、基础设施层等，使不同市场的业务逻辑能够独立演进。
- **限界上下文（Bounded Context）**：划分清晰的市场功能模块，确保各市场的业务逻辑互不干扰。
- **聚合（Aggregate）**：通过业务对象的合理归类，降低耦合度，提升数据一致性管理。


# 2 多市场功能特性管理

在以上三类功能特性中，**业务功能特性** 已经沉淀为领域模型的一部分，与领域本身深度绑定，因此无需额外的特殊管理或维护。

基于此，我们重点针对 **市场差异化功能特性**（通常以业务 Hardcode 形式存在）和 **动态功能开关（Feature Flag）** 进行规划与设计，以确保跨区域电商系统的灵活性和可扩展性。

## **2.1 调研和规划**

- 基于Monorepo项目的代码管理和复用策略，我们将“**动态功能开关（Feature Flag）”** 归类至 **基础设施层**，并存放在 `joyboy/packages/monorepo-features` 目录下，以便多个项目共享复用，实现统一的功能开关管理。
- 对于“**市场差异化功能特性”**，由于其本质上属于**领域内的业务模型**，应归属**领域层**`joyboy/libs/modules/order/domain`中进行管理，以确保业务逻辑的清晰性和可维护性。
- 同时，基于 **DDD 设计规范**，为了避免直接在领域模型中引用 **monorepo-features**（从而污染领域模型），我们在`joyboy/libs/modules/order/services）`引入**domainFeatureService** 作为适配层，负责在 **领域层** 与 **功能开关** 之间建立抽象接口，统一向领域内暴露 Feature API。
```
领域层 (Domain Layer)
 ├── MarketStrategy (市场策略接口)
 ├── USMarket, EUMarket, SGMarket, CAMarket (具体市场实现)
 ├── DomainFeatureService (统一 Feature API 访问)
 └── MarketFactory (市场策略工厂)
 
基础设施层 (Infrastructure Layer)
 ├── monorepo-features (Feature Flag 管理)
 └── featureManager (功能开关管理)
```
