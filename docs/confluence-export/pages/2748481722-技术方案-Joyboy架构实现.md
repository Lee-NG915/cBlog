现在初始化问题解决的不是很好RSC的限制问题SSR下的序列化处理方式序列化后的性能影响> - **No global stores** - Because the Redux store is shared across requests, it should not be defined as a global variable. Instead, the store should be created per request.
- **RSCs should not read or write the Redux store** - RSCs cannot use hooks or context. They aren't meant to be stateful. Having an RSC read or write values from a global store violates the architecture of the Next.js App Router.
- **The store should only contain mutable data** - We recommend that you use your Redux sparingly for data intended to be global and mutable.

```
sequenceDiagram
    actor Client
    participant CDN
    participant Server
    participant API

    Note over Client,Server: 首次请求
    Client->>Server: 发起首次请求
    Server->>API: 请求初始数据
    API-->>Server: 返回数据
    Server->>Server: 渲染服务端组件和客户端组件
    Server->>Server: 生成初始HTML
    Server-->>Client: 返回初始HTML
    Client->>Client: 渲染初始HTML
    
    Note over Client,CDN: 加载客户端组件
    Client->>CDN: 请求客户端组件JS
    CDN-->>Client: 返回客户端组件JS
    Client->>Client: 执行客户端组件JS
    Client->>Client: Hydration过程

    Note over Client,Server: 后续交互
    
    Note over Client,Server: 1. 仅客户端状态更新
    Client->>Client: 更新客户端状态
    Client->>Client: 重新渲染受影响的客户端组件

    Note over Client,Server: 2. 客户端API调用
    Client->>API: 直接请求数据
    API-->>Client: 返回数据
    Client->>Client: 更新状态和UI

    Note over Client,Server: 3. Server Action调用
    Client->>Server: 触发Server Action
    Server->>API: 请求/更新数据（如需要）
    API-->>Server: 返回数据
    Server->>Server: 执行action逻辑
    Server->>Server: 更新服务端状态（如需要）
    Server-->>Client: 返回action结果
    Client->>Client: 更新客户端状态
    Client->>Client: 重新渲染相关组件

    Note over Client,Server: 4. RSC更新（通过Server Action触发）
    Client->>Server: 触发需要RSC更新的Server Action
    Server->>API: 请求更新所需数据
    API-->>Server: 返回数据
    Server->>Server: 执行action逻辑
    Server->>Server: 更新服务端组件
    Server->>Server: 生成RSC Payload
    Server-->>Client: 返回action结果和RSC Payload
    Client->>Client: 解析RSC Payload
    Client->>Client: 更新组件树
    Client->>Client: 重新渲染受影响的组件

    Note over Server,Client: RSC Payload 结构
    Note over Server,Client: {<br/>tree: [更新的组件树结构],<br/>服务端组件: [更新的服务端渲染结果],<br/>客户端组件: [更新的引用和props]<br/>}

    Note over Server: 服务端组件：在服务器渲染
    Note over Client: 客户端组件：在客户端渲染
    Note over Server: Server Actions：在服务器执行，可能触发RSC更新
```
