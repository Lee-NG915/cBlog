## 1.  整体架构

### 1.1 模块结构

```
Account/
├── index.js                    # 主容器，路由导航
├── config.js                   # 优惠券类型配置
├── ProfileNew/                  # 个人资料管理子模块
├── Orders.js                   # 订单历史页面
├── OrderDetails.js             # 订单详情页面
├── Vouchers.js                 # 优惠券管理页面
├── Address.js                  # 地址管理页面
├── Reviews.js                  # 评价管理页面
├── ConvertToCredits.js         # 积分转换组件
├── ItemList.js                 # 订单商品列表组件
├── ReviewItem.js               # 评价项组件
├── ShipmentStatus.js           # 物流状态组件
└── style.scss                  # 样式文件
```

### 1.2 技术依赖

- **状态管理**: Redux + React-Redux
- **路由**: React Router v3
- **UI框架**: @castlery/fortress + 自定义SCSS
- **API客户端**: 自定义 ApiClient (基于superagent)
- **响应式**: 自定义 useBreakpoints Hook
## 2. 页面分析

### 2.1 主导航页面 (index.js)

- 侧边栏导航菜单 (桌面端) / 顶部Tab导航 (移动端)
- 基于URL路径的页面切换
- 条件性渲染Yotpo奖励功能
- 响应式布局适配
**导航菜单项:**

```
pages = [
  { url: ['/account/profile'], name: 'Account', icon: 'avatar' },
  { url: ['/account/orders', '/account/order-details'], name: 'Orders', icon: 'shop-cart-outline' },
  { url: ['/account/vouchers'], name: 'Vouchers', icon: 'voucher' },
  { url: ['/account/rewards'], name: 'Rewards', icon: 'rewards' }, // 条件性渲染
  { url: ['/account/address'], name: 'Address', icon: 'address-book' },
  { url: ['/account/reviews'], name: 'Reviews', icon: 'star-outline' }
]
```

> Rewards Page 为 StoryBlok配置，项目中仅实现了页面中的弹窗提示

本地状态: `onShowTips` (控制生日奖励提示显示)

- 订单历史列表展示

```

manualDiscountTotal = line_items.reduce((result, item) =>

### 2.4 Voucher Page (Vouchers.js)

- 优惠券列表展示

- 优惠券详情工具提示
**业务逻辑:**

```
// 可转换优惠券筛选
convertibleVouchers = vouchers.filter(item => item.can_convert);
total = convertibleVouchers?.length * 50; // 每张50积分

// 转换条件
convertValid = __YOTPO_ENABLED__ && isBefore('2023-01-01 00:00');

**优惠券类型格式化:**

- 使用 `config.js` 中的 `formatPreferences` 函数

## 3.  API 接口整理

### 3.1 Redux Modules API

| 模块 | **api** | 描述 |
| auth | `/users/me` | 更新用户信息 |
| auth | `/users/me` | 获取当前用户 |
| order | `/orders` | 获取订单列表 |
| voucher | `/vouchers` | 获取优惠券列表 |
| subscription | `/subscription` | 获取订阅设置 |

| address | `/addresses` | 获取地址列表 |
| address | `/addresses` | 添加地址 |
| address | `/addresses/:id` | 删除地址 |


### 3.2 直接API调用

| 文件 | 接口 | 方法 | **api** | 描述 |
| OrderDetails.js | 获取订单详情 | GET | `/orders/${number}` | 根据订单号获取详情 |
| Vouchers.js | 优惠券转积分 | POST | `/users/me/vouchers/exchange` | 将优惠券转换为积分 |
| ProfileNew/RestPwdLink.tsx | 发送重置链接 | POST | `/passwords/reset` | 发送密码重置邮件 |
| Reviews.js | 获取评价列表 | GET | `/reviews` | 获取用户评价 |


## 4. 埋点追踪分析

### 4.1 TrackableLink 埋点

**使用位置:**

```
// 主导航菜单 (index.js)
<TrackableLink 
  path={this.pages[key].url[0]} 
  menuType="user_menu" 
  text={this.pages[key].name}
>

// 登出按钮
<TrackableLink 
  menuType="user_menu" 
  text="Log Out" 
  onClick={logout} 
  isOriginal
>
```

**埋点参数:**

- `menuType`: "user_menu" (菜单类型标识)
- `text`: 具体菜单项名称
- `path`: 跳转路径
### 4.2 其他追踪点

**页面访问追踪:**

- 使用 `Helmet` 组件进行页面路径追踪
- 每个子页面都有路径标识
**用户行为追踪:**

- 优惠券转换操作
- 订单查看行为
- 地址管理操作
- 个人信息修改
## 5.  现有问题

**代码层面:**

- 文件结构混乱
- 样式混乱（行内 / 类 / 全局）
- 复杂的重定向逻辑（reward page 与 the-castlery-club的逻辑不够清晰）
**架构层面:**

- Redux 样板代码较多
- API 客户端封装不统一
- 错误处理机制不一致
