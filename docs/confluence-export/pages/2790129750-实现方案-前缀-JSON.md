## 配置获取实现

demo 分支: CU-86epvh16x

> 由于当前太多的业务代码是通过EcEnv获取环境变量, 并且这些代码都会应用于pos/web, 所以需要通过`NEXT_PUBLIC_CHANNEL` 来区分是使用pos / web 的配置.

具体实现可查看 joyboy/libs/config/src/values

1. - 所有基础配置都配置于: `libs/config/src/values` 下.

1. - 提供获取方法 `config(paths)` 

```
// config func

import values from './values'; // json 配置

/**
 *
 * @param path {string | Array<string>} - The path to the config value
 * @returns {any} - The config value
 *
 * @example Config('gtmID') // 'GTM-W4XWZH8'
 * @example Config(['US', 'currency']) // get US.currency => 'USD'
 * @example Config('US.currency') // get US.currency => 'USD'
 */

export default function getConfig(path: string | Array<string>): any {
  const pathArray = Array.isArray(path) ? path : path.split('.');
  if (pathArray.length === 0) {
    throw new Error('You must provide a path to get config value');
  }
  let result: ConfigValues = values;
  if (result === undefined) {
    throw new Error('Config values not found');
  }
  for (const key of pathArray) {
    if (result[key] === undefined) {
      throw new Error(`Config value not found for path: ${path}`);
    }
    result = result[key];
  }
  return result;
}
```

```
import {EcEnv} from '../ec-env';

// common-values.ts 所有国家都一样的ENV
const common_values = {
  GTM_ID: EcEnv.GTM_ID,     
  API_HOST: EcEnv.API_HOST, 
  GOOGLE_MAP_API_KEY: "AIzaSyAKmcMW98vJa4RtQ06RNEBjvCR66IpFSGE",
  // 构建配置
  build: {                 
    output: "standalone",
    assetPrefix: process.env.NODE_ENV === 'production' ? process.env.ASSETS_PATH : '';
  }
}

// sg-values.ts
const sg_values = {
  currency: "SGD",
  TIME_ZONE: "Asia/Singapore",
  SOMEVALUE: EcEnv['SG.xxxxx'],                 // 在ENV中使用前缀区分
  CLIENT_VALUE: EcEnv['NEXT_PUBLIC_SG.xxxxx'], // 在ENV中使用前缀区分
}

// au-values.ts
const au_values = {
  currency: "AUD",
  TIME_ZONE: "Asia/Singapore",
  SOMEVALUE: EcEnv.[AU.SOMEVALUE], // 在ENV中使用前缀区分
}
```
> 以上使用到ENV时, 在客户端也需要获取时, 在nextjs中需要加上`NEXT_PUBLIC_` 前缀.
