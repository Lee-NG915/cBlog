## 何时要写设计文档

- 该项目将需要至少一个月的工程时间。
- 这一变更将对软件的扩展和维护产生长期的影响。
- 添加或者更新新的依赖
- 该变更将显著影响其他团队。
## 行为准则

| 需要做的 | 不应该做的 |
| 使用设计文档模板 | 在意早晚会变的实验性的代码 |
| 阅读博客、论文和一些演讲文稿来获取灵感 | 只讨论一项解决方案 |
| 对于你看到的一且保持批判性思考 | 让非母语阻止你写作 |
| 在设计结构就编写实验性的代码 | 在具体实施方案和计划有些偏离时就忘记更新设计文档 |
| 学会清晰地写作，并经常练习 | 消极地参与团队设计讨论 |
| 对设计文档进行版本控制 |  |
| 对队友的设计提出问题 |  |
