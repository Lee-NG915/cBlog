# 前端多区域特性标记系统设计

## 1. 系统概述

特性标记（Feature Flags）系统允许我们在不同区域灵活地开启或关闭功能，而无需为每个区域单独构建代码。这种方案可以显著提升开发效率并简化部署流程。

## 2. 系统架构

### 2.1 基础架构

```
src/
├── features/
│   ├── config/
│   │   ├── features.ts       # 特性配置定义
│   │   └── regions.ts        # 区域定义
│   ├── components/           # 特性相关组件
│   ├── hooks/               # 特性标记 hooks
│   └── types/               # 类型定义
├── pages/
└── services/
```

### 2.2 核心组件

1. - 特性配置定义
2. - 特性检查 hooks
3. - 特性包装组件
4. - 区域配置管理
## 3. 详细实现

### 3.1 特性配置定义

```
// src/features/config/regions.ts
export const REGIONS = {
  US: 'US',
  CN: 'CN',
  EU: 'EU',
} as const;

export type Region = keyof typeof REGIONS;

// src/features/config/features.ts
export const FEATURES = {
  INSURANCE: 'insurance',
  PAYMENT: 'payment',
  SOCIAL_LOGIN: 'socialLogin',
} as const;

export type Feature = keyof typeof FEATURES;

export const FEATURE_CONFIG = {
  [FEATURES.INSURANCE]: {
    enabledRegions: [REGIONS.US, REGIONS.EU],
    defaultEnabled: false,
    description: 'Insurance service feature',
  },
  [FEATURES.PAYMENT]: {
    enabledRegions: [REGIONS.US, REGIONS.EU, REGIONS.CN],
    defaultEnabled: true,
    description: 'Payment processing feature',
  },
  [FEATURES.SOCIAL_LOGIN]: {
    enabledRegions: [REGIONS.CN],
    defaultEnabled: true,
    description: 'Social media login feature',
  },
} as const;
```

### 3.2 特性检查 Hooks

```
// src/features/hooks/useFeature.ts
import { useCallback } from 'react';
import { FEATURE_CONFIG, Feature } from '../config/features';

export function useFeature(featureName: Feature) {
  const currentRegion = process.env.NEXT_PUBLIC_REGION;
  
  const isEnabled = useCallback(() => {
    const feature = FEATURE_CONFIG[featureName];
    if (!feature) return false;
    
    // 检查是否在启用区域列表中
    return feature.enabledRegions.includes(currentRegion) && feature.defaultEnabled;
  }, [featureName]);

  return {
    isEnabled: isEnabled(),
    featureConfig: FEATURE_CONFIG[featureName],
  };
}
```

### 3.3 特性包装组件

```
// src/features/components/FeatureGate.tsx
import { ReactNode } from 'react';
import { Feature } from '../config/features';
import { useFeature } from '../hooks/useFeature';

interface FeatureGateProps {
  feature: Feature;
  children: ReactNode;
  fallback?: ReactNode;
}

export function FeatureGate({ feature, children, fallback = null }: FeatureGateProps) {
  const { isEnabled } = useFeature(feature);
  
  if (!isEnabled) {
    return fallback;
  }
  
  return <>{children}</>;
}
```

### 3.4 使用示例

```
// src/pages/services.tsx
import { FeatureGate } from '@/features/components/FeatureGate';
import { FEATURES } from '@/features/config/features';
import InsuranceService from '@/components/InsuranceService';

export default function ServicesPage() {
  return (
    <div>
      <h1>Our Services</h1>
      
      <FeatureGate 
        feature={FEATURES.INSURANCE}
        fallback={<p>Insurance services are not available in your region.</p>}
      >
        <InsuranceService />
      </FeatureGate>
    </div>
  );
}
```

### 3.5 动态导入优化

```
// src/components/InsuranceService/index.tsx
import dynamic from 'next/dynamic';
import { useFeature } from '@/features/hooks/useFeature';
import { FEATURES } from '@/features/config/features';

const InsuranceComponent = dynamic(
  () => import('./InsuranceComponent'),
  { 
    ssr: false,
    loading: () => <div>Loading...</div>
  }
);

export default function InsuranceService() {
  const { isEnabled } = useFeature(FEATURES.INSURANCE);
  
  if (!isEnabled) {
    return null;
  }
  
  return <InsuranceComponent />;
}
```

## 4. CI/CD 配置

### 4.1 构建配置

```
# .github/workflows/ci.yml
jobs:
  build:
    runs-on: ubuntu-latest
    steps:
      - name: Build
        run: |
          # 只需要一次构建
          npm run build
          
      - name: Upload to all regions
        run: |
          VERSION=${GITHUB_SHA::8}
          # 并行上传到所有区域
          for region in us-east-1 ap-east-1 eu-west-1; do
            aws s3 sync .next/static s3://$region-bucket/static/$VERSION/ \
              --region $region &
          done
          wait
```

### 4.2 部署配置

```
# deployments/apps/nextjs/us-east-1/values.yaml
env:
  NEXT_PUBLIC_REGION: "US"
  NEXT_PUBLIC_API_URL: "<https://api.us.example.com>"
  NEXT_PUBLIC_ASSET_PREFIX: "<https://us-bucket.s3.amazonaws.com/static/${VERSION}>"
```

## 5. 最佳实践

### 5.1 特性标记命名规范

- 使用大写字母和下划线
- 名称应该清晰表达功能
- 添加适当的描述
### 5.2 代码组织

- 相关功能代码放在一起
- 使用类型系统确保类型安全
- 提供合适的 fallback 内容
### 5.3 性能优化

- 使用动态导入
- 实现代码分割
- 缓存特性检查结果
### 5.4 测试策略

```
// src/features/__tests__/useFeature.test.ts
describe('useFeature', () => {
  it('should enable feature for correct region', () => {
    process.env.NEXT_PUBLIC_REGION = 'US';
    const { result } = renderHook(() => useFeature(FEATURES.INSURANCE));
    expect(result.current.isEnabled).toBe(true);
  });

  it('should disable feature for incorrect region', () => {
    process.env.NEXT_PUBLIC_REGION = 'CN';
    const { result } = renderHook(() => useFeature(FEATURES.INSURANCE));
    expect(result.current.isEnabled).toBe(false);
  });
});
```

## 6. 维护和更新

### 6.1 添加新特性

1. - 在 `features.ts` 中添加特性定义
2. - 配置启用区域
3. - 添加相关组件和逻辑
4. - 更新测试用例
### 6.2 更新特性配置

1. - 修改 `FEATURE_CONFIG`
2. - 确保所有相关组件都正确处理新配置
3. - 运行测试确保没有破坏现有功能
### 6.3 移除特性

1. - 标记特性为废弃
2. - 在适当的时机完全移除代码
3. - 更新文档和测试
## 7. 监控和分析

建议实现以下监控指标：

- 特性使用率
- 性能影响
- 错误率
- 用户反馈
## 8. 总结

特性标记系统能够：

1. - 简化多区域部署
2. - 提高代码维护性
3. - 支持灵活的功能控制
4. - 减少构建和部署复杂性
通过合理使用特性标记，我们可以用单一代码库和单次构建来满足多区域的不同需求。
