# addx.ai / 积加科技公司研究

> 调研日期：2026-07-23  
> 调研范围：公司身份、主体关系、团队、融资、产品、市场、技术、招聘及公开社区讨论。  
> 证据原则：优先使用官网、应用商店、投资方、监管/知识产权文件等一手资料；招聘文案视为公司自述；匿名面经和用户评论只作为未经核实的社区信号。

## 一页结论

addx.ai / A4x 对应的目标公司，核心国内主体是 **北京积加科技有限公司**，成立于 2018 年 12 月，创始人兼 CEO 为蒲立。公司早期定位是面向摄像头厂商和品牌商的全球化 AIoT PaaS/SaaS 平台，提供硬件方案、嵌入式软件、App、P2P 音视频、云存储和视觉 AI 服务。近年的招聘材料显示，公司正在把技术叙事扩展到多模态模型、视觉记忆、Agent、3D 重建和世界模型，但这些新方向目前主要由招聘信息和专利线索支持，公开商业化证据有限。

最明确的消费端产品是 **VicoHome**：它连接智能摄像头，提供直播、回看、通知、云存储和人/车/包裹/宠物识别订阅。2026 年又上线了面向观鸟场景的 **Vico Nature**。VicoHome 在 Google Play 显示 100 万以上下载，在美国 App Store 显示约 11.1 万条评分；这能够支持“拥有较大规模海外用户”的判断，但不能独立证明招聘文案中的“全球数百万用户”。

公司有真实的全球化业务证据：海外应用商店、多语言 App、AWS Marketplace 服务、Alexa 相关能力、海外商标与应用发行活动，以及深圳、北京、杭州等地的招聘。公开招聘标签通常将其列为 B 轮、100–499 人，但人数和资本数据在不同平台间存在冲突，不应当作审计数据。

对候选人最重要的判断是：这不是单纯的企业官网岗位。结合公开产品和招聘描述，Web/全栈岗位很可能服务于全球官网、营销与电商站点、用户 Web 页面、订阅/增值服务或内部运营平台。HR 提到 Shopify 与这一商业场景相容，但本次公开检索没有找到能够独立验证 Shopify 技术栈的第一方页面，应在面试中确认具体站点、后端边界和岗位所有权。

## 3. 业务演进

### 2018–2019：以摄像头切入 AIoT 平台

2019 年，公司对外定位为全球化 AIoT PaaS + Service 平台，向摄像头品牌商和制造商提供 AI 边缘计算、视频云、客户端 App 与硬件方案。经纬创投称，公司成立不到一年已建立覆盖数十种场景的 AI 技能库，并有第一款摄像头方案通过客户量产进入全球市场。

### 2020–2022：A4x 平台、VicoHome 与智能安防订阅

A4x 官网展示了较完整的平台形态：

- 嵌入式软件、App 开发、IoT Essence 和运营服务。
- All-in-One App、OEM App 和 App SDK 三种交付方式。
- 摄像头、门铃、云存储、视觉智能、智能语音与设备联动。
- 面向厂商、开发者和品牌商的一站式方案。

VicoHome 将这些能力直接面向消费者：直播、回看、双向通话、运动通知、云存储和 AI 分类提醒，并通过 Awareness 订阅收费。

公开专利线索显示，公司继续围绕安防事件、上下文视频生成、鸟类识别、视频编解码和多模态标注研发。2025 年公司招聘材料开始集中强调多模态大模型、长时记忆、Agent、AIGC、3D 重建和 World Model；2026 年上线 Vico Nature，形成自然观察场景的独立 App。

### 对 Web/全栈岗位的推断

公开产品决定了 Web 工程可能涉及四类系统：

1. 全球官网、产品站、SEO 与多语言内容。
2. 面向用户的账户、设备、订阅、订单或支持页面。
3. 面向品牌商/制造商的 A4x 控制台、OEM 配置和运营后台。
4. 面向消费硬件的独立站、电商、营销和增值服务转化。

这是依据产品形态做出的合理推断，不代表该岗位一定覆盖全部范围。公开网页没有检索到 Shopify、Hydrogen、Liquid 或具体 Node 后端框架的可靠一手证据。面试时应确认：

- Shopify 是自有 DTC 商城、多个品牌店铺，还是客户代运营项目？
- 岗位主要写 Liquid/Theme、Hydrogen/Remix，还是 React + Shopify Admin/Storefront API？
- “全栈”是 Node BFF、小型业务服务，还是需要承担数据库、队列、部署和 on-call？
- 官网、商城、VicoHome Web、A4x 控制台分别由哪个团队负责？

## 11. 风险与待确认事项

### 高优先级

1. **岗位边界不清**：招聘名义为前端，但 HR 沟通为 Shopify 全栈。需要确认前端、Node、Shopify 配置、支付/订单和基础设施各占多少。
2. **业务线归属不清**：岗位可能属于官网/品牌商城，也可能服务某个尚未公开的新硬件品牌。
3. **AI 新方向成熟度不清**：多模态、Agent 和 World Model 是正式产品路线、研究团队，还是招聘品牌叙事，需要询问现阶段用户、数据和交付物。
4. **组织和工作方式缺少公开证据**：没有足够可靠的员工讨论可以判断加班、绩效、晋升和流动性。
5. **海外主体关系不透明**：Photon Sail、VICO TECH、历史 Addx Hong Kong 与北京积加的股权和职责关系没有公开完整说明。

### 面试中值得反向确认

- 这个岗位入职后 3 个月和 6 个月的成功标准是什么？
- 当前 Shopify 店铺数量、目标国家、主要品牌和 GMV 阶段是什么？
- 使用 Shopify Theme/Liquid 还是 Hydrogen/Remix？是否有 Node BFF 和自建数据库？
- 支付、税务、物流、订阅、账号和设备绑定分别由 Shopify 还是自研服务负责？
- 多语言内容如何管理，是否使用 Shopify Markets、CMS、翻译平台和 `hreflang`？
- Core Web Vitals 的当前基线、监控方式和主要瓶颈是什么？
- Web 团队人数、前后端比例、代码评审、测试、发布和 on-call 机制是什么？
- VicoHome/Vico Nature 与 Shopify 商城之间是否有账号、设备、订阅或售后数据打通？
- 公司所说的“数百万用户”对应注册量、设备连接数、月活还是累计用户？
- 多模态/Agent/World Model 当前最接近商业化的是哪个产品，Web 团队会怎样参与？

## 12. 最终判断

| 判断 | 结论 | 置信度 |
| --- | --- | --- |
| 公司真实存在且持续运营 | 北京积加、深圳积加、A4x 官网、App 和持续招聘互相印证 | 高 |
| 核心业务是视觉 AIoT 与智能摄像头 | 官网、App、AWS Marketplace、专利和历史融资材料一致 | 高 |
| 有较强海外业务基础 | 多语言 App、应用商店规模、AWS/Alexa、海外主体和招聘口径 | 高 |
| 服务全球数百万用户 | 有 100 万+ Android 下载和大量 iOS 评分支撑，但口径未审计 | 中 |
| 已形成成熟的世界模型/Agent 产品 | 目前主要是招聘与研发线索 | 低至中 |
| 公司为 B 轮、约 100–499 人 | 多个招聘平台如此标注，但不是官方披露 | 中 |
| Web 岗位主要使用 Shopify | 来自 HR 沟通；公开互联网未找到独立验证 | 低至中 |
| 社区普遍认为加班严重或文化有问题 | 没有足够可核验样本支持 | 低，不应下结论 |

总体上，这是一家从视觉 AIoT 平台和智能安防起步、已经拥有真实海外消费端产品与订阅业务、正在向自然观察和多模态 AI 扩展的成长型软硬件公司。它对前端候选人的潜在价值在于全球化、性能、电商、订阅、设备生态和 AI 产品结合；主要风险则是岗位边界、组织成熟度、新业务阶段和工作负荷缺少公开透明度，需要通过面试反向确认。
