# 07 · 前端基础八股（手撕 + 概念）

按真题情报：**带 TTL 的 LRU 最高优先**；其余按手撕 → JS → React → 浏览器 → 网络 → TS → 算法。

---

## 手撕：带 TTL 的 LRU（该公司真题）

结构：

- `Map`：O(1) 查节点。  
- **双向链表**：维护 recency（头最旧 / 尾最新，或相反，说清即可）。  
- 节点：`key, value, expireAt`。

操作：

- `get`：不存在或过期 → 删并返回空；命中 → 移到最新端。  
- `put`：已存在则更新值与过期、移到最新；新 key 超容量则淘汰最旧。  
- 容量：只计未过期有效条目（或 get/put 时惰性删）。

TTL：

- **惰性过期**：读写时检查 `now > expireAt`（主路径，必讲）。  
- **定时清理**：可选扫链表/桶，防大量冷 key 占内存。  
- 过期算未命中，不延长 TTL（除非题目要求 sliding）。

口述顺序：需求 → 结构选型 → get/put → 过期策略 → 复杂度 O(1) → 边界（capacity=0、expire=0）。

## 其它手撕要点

### 防抖 / 节流

- 防抖：停下来才执行（搜索输入）。  
- 节流：固定间隔最多一次（滚动）。  
- 注意：`this`、参数、leading/trailing、cancel。

### 深拷贝

- 手写：递归 + `WeakMap` 处理循环引用。  
- 类型：Date/RegExp/Map/Set/Array；函数/DOM 通常不拷或按需求。  
- `structuredClone` / JSON 局限（提一句）。

### Promise.all / allSettled

- `all`：全成才成，一败即败。  
- `allSettled`：等全部结束，返回 status 列表。  
- 手写 all：计数 + 按序填数组。

### call / apply / bind

- 改 `this`；apply 参数数组；bind 返回绑定函数（可柯里化）。  
- 手写 bind：保存 args，`new` 时忽略绑定 this。

## JS 核心

### 事件循环

- 调用栈 → 微任务（Promise/queueMicrotask）→ 宏任务（setTimeout、I/O）。  
- 每宏任务后清空微任务队列。  
- Node 与浏览器阶段差异了解即可，面试画浏览器模型足够。

### 闭包

- 函数引用外层词法变量；缓存、柯里化、hooks 过期闭包陷阱。

### 原型链

- 实例 → `__proto__` → prototype → … → null。  
- `new` 过程；class 语法糖。

### this

- 默认/隐式/显式/new/箭头（词法 this）。  
- 与 bind 手撕联动。

## React

### Fiber

- 可中断单元；reconcile 与 commit 分离；支持优先级/并发。  
- 双缓存思想（能点到即可）。

### Hooks

- `useState` 快照；批更新。  
- `useEffect`：依赖比较、清理函数、别当同步渲染用。  
- 闭包陷阱：过期 state → 函数式更新或正确依赖。  
- 规则：顶层调用、依赖诚实。

### 并发特性

- 可中断渲染、useTransition/useDeferredValue 降紧急度。  
- 与 INP：减少紧急更新阻塞输入。

### RSC（与项目打通）

- 默认 Server；`'use client'` 边界；少下发 JS；数据在服务端取。  
- hydration 只针对 client 子树。

---

## 浏览器

### 渲染流程

- DOM/CSSOM → Render Tree → Layout → Paint → Composite。  

### 重绘 / 回流

- 回流改几何；重绘改外观；合成层优化。  
- 批读批写、避免强制同步布局。

### 缓存（与 ISR/Redis 打通）

- 强缓存：Expires / Cache-Control。  
- 协商：ETag / Last-Modified → 304。  
- CDN：边缘命中降延迟；与「多 Pod 共享应用缓存」分层讲：浏览器/CDN/应用 Redis 各一层。

### 跨域

- 同源策略；CORS 预检；简单/非简单请求。  

### XSS / CSRF

- XSS：转义、CSP、HttpOnly。  
- CSRF：SameSite、CSRF Token、不滥用 Cookie 鉴权于危险操作。

---

## 网络

- HTTP/2：多路复用、头部压缩；减少队头阻塞（相对 HTTP/1.1）。  
- HTTPS：TLS 握手、证书；中间人防护。  
- CDN：就近、缓存、回源；全球化站点结合多市场部署经验讲。  
- DNS → TCP/TLS → 请求 → 响应。

## TypeScript

- 泛型：参数化类型，约束 `extends`。  
- 条件类型：`T extends U ? X : Y`。  
- `infer`：在条件类型里推断。  
- 工具类型：`Partial` `Pick` `Omit` `Record` `ReturnType` `Awaited`。  
- `unknown` vs `any`；类型收窄。

## 算法：合并区间（该公司真题相关）

- 先按起点排序。  
- 扫一遍：能合并则扩 end，不能则推入结果。  
- 复杂度 O(n log n)。  
- 变体：插入区间、相交判断——同一套排序扫描思路。
