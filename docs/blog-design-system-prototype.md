# Color Blog 个人博客设计系统原型

## 1. 结论

当前布局不算错误，也符合个人知识库类博客的基础实践：左侧固定导航、中心内容流、文章详情聚焦阅读。但它更像一个通用后台/文档站模板，缺少个人博客应有的“作者气质”和内容主张。

参考站点的共性不是某个具体组件，而是更清晰的品牌判断：

- Finiam Blog 强调技术团队的轻松表达，文章列表直接、信息密度高，但不依赖复杂装饰。
- Fictiv Journal 把博客当成品牌杂志，开场文案更有态度，留白和大标题强化审美。
- PayPro Global Blog 更偏商业内容门户，突出精选文章、分类和转化入口。
- ElevenLabs Blog 更像产品与研究发布台，使用克制的大面积留白、清晰分类、强标题层级。

Color Blog 应定位为：**一个产品分析与工程实践并重的个人研究笔记站**。视觉上不追求炫技，而要呈现“冷静、结构化、有判断力”的个人风格。

## 2. 设计原则

### 2.1 内容优先

页面首屏不要只展示统计数字，要优先回答：

- 这个人长期关注什么？
- 最近在研究什么？
- 哪篇文章最值得读？

首页的核心不是“有多少文章”，而是“我为什么要读你写的东西”。

### 2.2 个人风格可识别

当前 UI 的主要问题是所有卡片都长得像 Tailwind 示例。新的设计系统需要建立三个识别点：

- 语气：用“产品分析札记 / 工程实践 / 生活观察”替代泛化的分类表达。
- 版式：减少均匀卡片堆叠，增加 Editorial Hero、Featured Note、Reading Path。
- 视觉：使用温暖纸张背景、墨色正文、克制蓝绿色强调色，形成“研究笔记”气质。

### 2.3 PC 优先但不呆板

既然项目当前只考虑 PC 端，应充分利用横向空间。保留侧栏，但弱化它的存在感，让内容区更像一张可阅读的编辑版面，而不是应用后台。

## 3. 品牌方向

### 3.1 品牌关键词

- Calm：安静、留白、低噪音。
- Analytical：结构化、有数据和判断。
- Personal：不完全公司博客化，保留作者视角。
- Editorial：像一本持续更新的研究杂志。

### 3.2 站点一句话

记录产品分析、工程实践和增长思考，把复杂问题拆成可以行动的判断。

### 3.3 首页主标题建议

不要只写“Color 的个人博客”，建议升级为带观点的标题：

> Color's Field Notes on Product, Growth and Engineering

中文副标题：

> 用产品分析和工程实践，记录问题如何被拆解、验证和落地。

## 4. 视觉系统

### 4.1 色彩

| Token | 用途 | 建议值 | 说明 |
| --- | --- | --- | --- |
| `paper` | 页面背景 | `#F7F4ED` | 比纯白更像纸张，降低模板感 |
| `surface` | 卡片背景 | `#FFFDF8` | 保持阅读干净 |
| `ink` | 主文字 | `#1F2933` | 稳重但不死黑 |
| `muted` | 次级文字 | `#687076` | 元信息、摘要 |
| `line` | 边框 | `#E5DFD3` | 更温暖的分割线 |
| `accent` | 重点链接/标签 | `#0F766E` | 蓝绿色，适合分析与工程气质 |
| `accent-soft` | 标签背景 | `#E6F3F1` | 轻量强调 |

深色模式可保留，但不应成为主风格。先把浅色模式做到有识别度。

### 4.2 字体

建议使用系统字体，但建立明确层级：

- 标题：`font-semibold` 或 `font-bold`，字距略收紧。
- 正文：17px，行高 1.8。
- 元信息：12-13px，全部使用同一套 muted 色。
- 代码：继续使用等宽字体，但背景降低对比，避免满篇灰块抢注意力。

### 4.3 圆角与边框

当前圆角和阴影偏通用。建议：

- 卡片圆角：`16px`
- 小标签圆角：`999px`
- 边框：使用暖灰，不用纯灰
- 阴影：尽量移除或只保留非常轻的 hover，不依赖阴影制造层级

## 5. 首页原型

### 5.1 页面结构

```text
┌────────────────────────────────────────────────────────────┐
│ Left Rail │ Hero: 当前研究主题 + 作者定位       │ Index     │
│           │ Featured Note                         │ Stats     │
│           │ Reading Paths                         │ Tags      │
│           │ Latest Essays                         │           │
└────────────────────────────────────────────────────────────┘
```

### 5.2 左侧导航

现状：像应用侧边栏。

建议：改成轻量目录栏。

- 顶部显示 `Color` 和一句短签名。
- 主导航保留：首页、文章、关于。
- 分类改名为“路径”，例如：产品分析、工程实践、生活观察。
- 底部保留主题切换和 GitHub，但降低视觉重量。

### 5.3 Hero 区

目标：让首页一打开就有个人表达，而不是统计面板。

内容建议：

- Eyebrow：`FIELD NOTES`
- H1：`把产品问题拆成可验证的判断`
- Description：`这里记录增长策略、数据分析、前端工程和长期学习。每篇文章都尽量回答一个真实问题。`
- CTA：`阅读最新札记`、`查看研究路径`

### 5.4 Featured Note

把最新或最重要的一篇文章放大展示，而不是所有文章卡片同权重。

字段：

- 分类
- 标题
- 摘要
- 3 个标签
- 发布日期
- 阅读时间

视觉：

- 宽卡片
- 左侧文字，右侧可以是简洁的几何/代码片段/数据摘要，不必依赖封面图

### 5.5 Reading Paths

替代现在三个普通分类卡。

每个路径包含：

- 名称
- 一句话说明
- 文章数
- 最近一篇文章

示例：

- 产品分析：从数据、用户行为和业务目标中形成产品判断。
- 工程实践：记录前端、部署、工具链和架构取舍。
- 生活观察：保留非工作语境下的长期思考。

### 5.6 Latest Essays

文章列表建议做成“杂志式列表 + 重点卡片”的混合：

- 第一篇大卡
- 后续文章用横向列表
- 右侧显示日期、分类、阅读时间

这样比 3 列等宽卡片更有节奏。

## 6. 文章详情页原型

### 6.1 页面结构

```text
┌────────────────────────────────────────────────────────────┐
│ Left Rail │ Back / Category / Date                         │
│           │ Title                                          │
│           │ Summary / Tags                                 │
│           │ Article Body                     │ On this page │
└────────────────────────────────────────────────────────────┘
```

### 6.2 文章头部

当前文章头部信息完整，但像普通卡片。建议：

- 去掉大面积卡片感，改为开放式标题区。
- 标题下增加 `excerpt` 或自动摘要。
- 分类标签只保留一个主分类，其余作为 tag。
- 发布日期、更新日期、阅读时间放在同一行。

### 6.3 正文阅读

正文应该是整个站点最强的部分。

建议：

- 正文宽度控制在 `720-780px`。
- 段落 17px，行高 1.8。
- H2 上方留足空间，形成章节节奏。
- 表格增加横向滚动、表头背景、数值右对齐。
- 代码块保留深色背景，但减少外层卡片边框。

### 6.4 文章侧栏

PC 端可以增加右侧窄栏：

- On this page：自动目录
- Article metadata：分类、标签、阅读时间
- Next：下一篇推荐

如果暂时不做自动目录，也可以先放一个“本文回答”的摘要盒子。

## 7. 组件规范

### 7.1 `EditorialHero`

用途：首页核心定位。

字段：

- eyebrow
- title
- description
- primaryAction
- secondaryAction

样式：

- 大标题 48-56px
- 背景不使用纯白，可以用 `paper` 上的开放区域
- 按钮只保留一个实心强调，另一个文字链接

### 7.2 `FeaturedPost`

用途：突出最新或精选文章。

字段：

- post
- reason，例如 `最新发布`、`推荐阅读`、`正在研究`

样式：

- 占两列宽
- 摘要最多三行
- 标签不超过三个

### 7.3 `ReadingPathCard`

用途：替代普通分类卡。

字段：

- name
- description
- count
- latestPost

样式：

- 不使用 icon 堆叠
- 用细线和留白体现结构

### 7.4 `EssayListItem`

用途：文章列表项。

字段：

- title
- excerpt
- category
- date
- readingTime

样式：

- 横向列表
- hover 只改变标题颜色和左侧细线

### 7.5 `ArticleShell`

用途：文章页布局容器。

组成：

- article header
- prose content
- optional right toc

## 8. 交互与动效

动效应克制：

- hover：颜色、边框、轻微位移即可。
- 不使用大面积阴影和复杂动画。
- 首页卡片 hover 不应让页面显得像电商卡片。
- 文章页优先稳定阅读，不做干扰性动画。

## 9. 信息架构建议

### 9.1 分类重命名

当前分类：

- 技术博客
- 学习日志
- 生活随记

建议改为更有个人定位的路径：

- 工程实践
- 产品分析
- 生活观察

如果后续文章增多，可增加：

- 数据笔记
- 工具与效率

### 9.2 文章 frontmatter 建议

```yaml
---
title: 增长策略基础
date: 2026-05-11
category: learning
tags:
  - Growth
  - GA4
  - Product Analytics
excerpt: 从收入目标出发，把增长问题拆成行为链路、用户特征和可行动的产品假设。
status: published
featured: true
---
```

### 9.3 首页排序策略

优先级：

1. `featured: true`
2. 最新发布
3. 有摘要的文章
4. 普通文章

## 10. 实施路线

### Phase 1：视觉基线

- 更新 Tailwind 色彩 token。
- 调整全局背景、卡片边框、正文宽度。
- 优化 `.prose` 排版、表格、代码块。

### Phase 2：首页重构

- 新建 `EditorialHero`。
- 新建 `FeaturedPost`。
- 将分类卡改为 `ReadingPathCard`。
- 将最新文章从 3 列卡片改为精选 + 列表。

### Phase 3：文章页重构

- 文章头部去卡片化。
- 增加摘要区。
- 调整正文宽度和右侧信息栏。
- 后续可增加自动目录。

### Phase 4：内容系统

- 增加 `featured` 字段。
- 完善每篇文章的 `excerpt`。
- 将文件组织和分类映射到新的“路径”语言。

## 11. 验收标准

首页验收：

- 首屏能表达作者定位，而不只是展示文章数量。
- 最新文章和精选文章有明显主次。
- 分类不再像普通后台导航，而像阅读路径。

文章页验收：

- 正文阅读宽度舒适。
- 表格、代码块、引用都有统一风格。
- 标题区能体现文章主题、分类和摘要。

品牌验收：

- 截图去掉文字后，仍能感觉它不是默认模板。
- 保留冷静、结构化、偏研究型的个人气质。
- 不牺牲阅读效率去做装饰。
