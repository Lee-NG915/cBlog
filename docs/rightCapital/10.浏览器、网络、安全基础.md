## cookie、session、token、JWT 的区别？

`cookie` 是浏览器存储并自动随请求携带的数据；`session` 是服务端保存登录状态，客户端通常只存 session id；`token` 是客户端携带的访问凭证；`JWT` 是一种自包含 token，里面有签名后的用户信息和过期时间。

## SameSite、HttpOnly、Secure 分别有什么作用？

`SameSite` 限制跨站请求携带 cookie，主要防 CSRF；`HttpOnly` 禁止 JS 读取 cookie，降低 XSS 后 token 被偷风险；`Secure` 要求 cookie 只能通过 HTTPS 发送。

## CORS 预检请求什么时候触发？

跨域请求不是简单请求时触发，比如用了 `PUT/PATCH/DELETE`、自定义 header、`Content-Type: application/json`、带复杂认证头等，浏览器会先发 `OPTIONS`。

## OPTIONS 请求的作用是什么？

用于询问服务端是否允许当前跨域请求的方法、header、origin。服务端返回允许后，浏览器才会发真正请求。

## 如何防 XSS？如何防 CSRF？

防 XSS：转义用户输入、避免 `dangerouslySetInnerHTML`、CSP、依赖安全审计、敏感 cookie 加 `HttpOnly`。  
防 CSRF：`SameSite` cookie、CSRF token、校验 Origin/Referer、关键操作要求用户态校验。

## localStorage、sessionStorage、cookie 的区别？

`localStorage` 持久保存，不随请求发送；`sessionStorage` 只在当前 tab 会话内有效；`cookie` 可设置过期时间，会自动随匹配请求发送，容量更小。

## HTTP 缓存有哪些头？

常见有 `Cache-Control`、`Expires`、`ETag`、`Last-Modified`、`If-None-Match`、`If-Modified-Since`、`Vary`、`Age`。

## ETag 和 Last-Modified 的区别？

`ETag` 基于资源内容或版本标识，精度更高；`Last-Modified` 基于最后修改时间，精度较粗，可能受时间粒度影响。

## GET、POST、PUT、PATCH、DELETE 如何取舍？

`GET` 查询资源；`POST` 创建资源或提交动作；`PUT` 整体替换资源；`PATCH` 局部更新资源；`DELETE` 删除资源。

## WebSocket 和 HTTP 有什么区别？

HTTP 是请求-响应模型，客户端发起一次请求服务端返回一次响应；WebSocket 是建立长连接后双向通信，适合实时消息、行情、协作场景。

## TCP 和 UDP 有什么区别？

TCP 面向连接、可靠、有序、带重传和拥塞控制；UDP 无连接、不保证可靠和顺序，但开销低、延迟小，适合音视频、游戏、DNS 等场景。

## 浏览器从输入 URL 到页面展示发生了什么？

解析 URL，查缓存，DNS 解析，建立 TCP/TLS 连接，发送 HTTP 请求，服务端返回 HTML，浏览器解析 HTML 构建 DOM，加载 CSS/JS/图片，构建 CSSOM 和 Render Tree，执行布局、绘制、合成，最后页面展示并完成 JS hydration/交互绑定。
