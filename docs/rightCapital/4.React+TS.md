## 你们用的是React哪个版本？项目中使用了哪些版本特性？现在最新的版本有了解过吗？

我们当前项目用的是 React 18，实际用到的主要是 React 18 这一代 的能力，而不是 19 的新 API。

比较典型的是：

- Suspense 做流式渲染和分段 fallback，PDP 上用得很多
- App Router 下的 Server Component / Client Component 拆分
- RSC 场景里的 cache()，做服务端去重和缓存复用
- 再配合 Next 的 useServerInsertedHTML 处理 Emotion 的 SSR 注入

如果说最新版本，我有关注过。
截至 2026-06-27，npm 上 react 最新 stable 我查到是 19.2.7，而官方最近一版主要功能更新我看的是 React 19.2。这一版我比较关注的点是 useEffectEvent、cacheSignal、<Activity />、Partial Pre-rendering，还有 SSR 下 Suspense reveal batching 的改进。

如果结合我们项目来讲，我觉得最有价值的不是盲目升级，而是优先评估：

- 和 Next 14 的兼容性
- 和我们现在的 RSC 边界、组件库、MUI Joy/Emotion 的兼容性
- 以及搜索、PDP、支付这些核心链路能不能真正吃到收益
