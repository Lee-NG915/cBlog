# 代码格式化指南

## 概述

本项目使用 Prettier 作为代码格式化工具，并配置了自动格式化功能。

## 自动格式化

### 保存时自动格式化

- 当你保存文件时，代码会自动使用 Prettier 进行格式化
- 这个功能已经在 `.vscode/settings.json` 中配置好了

### 粘贴时自动格式化

- 当你粘贴代码时，会自动进行格式化
- 这个功能已经在 `.vscode/settings.json` 中配置好了

## 手动格式化

### 快捷键

- `Shift + Alt + F`: 格式化当前文档
- `Ctrl + Shift + P`: 运行"Format All Files"任务（格式化所有文件）
- `Ctrl + Shift + C`: 运行"Check Format"任务（检查格式）

### 命令面板

1. 按 `Ctrl + Shift + P` 打开命令面板
2. 输入 "Tasks: Run Task"
3. 选择以下任务之一：
   - `Format All Files`: 格式化所有文件
   - `Check Format`: 检查格式（不修改文件）

### 终端命令

```bash
# 格式化所有文件
pnpm format

# 检查格式（不修改文件）
pnpm format:check
```

## 配置

### Prettier 配置

项目根目录的 `.prettierrc` 文件包含以下配置：

```json
{
  "semi": true,
  "trailingComma": "es5",
  "singleQuote": true,
  "tabWidth": 2,
  "useTabs": false,
  "printWidth": 120
}
```

### VS Code 设置

`.vscode/settings.json` 中配置了：

- 保存时自动格式化
- Prettier 作为默认格式化工具
- 保存时自动修复 ESLint 问题
- 保存时自动整理导入

## 支持的文件类型

以下文件类型支持自动格式化：

- TypeScript (`.ts`)
- TypeScript React (`.tsx`)
- JavaScript (`.js`)
- JavaScript React (`.jsx`)
- JSON (`.json`)
- CSS (`.css`)
- SCSS (`.scss`)
- HTML (`.html`)
- Markdown (`.md`)

## 故障排除

### 如果自动格式化不工作

1. 确保安装了 `esbenp.prettier-vscode` 扩展
2. 重启 VS Code
3. 检查文件类型是否在支持列表中
4. 确保文件没有语法错误

### 如果格式不符合预期

1. 检查 `.prettierrc` 配置
2. 运行 `pnpm format:check` 查看问题
3. 确保没有其他格式化工具冲突

## 最佳实践

1. **保存前格式化**: 养成保存前手动格式化的习惯
2. **使用快捷键**: 熟悉格式化快捷键，提高效率
3. **定期检查**: 定期运行 `pnpm format:check` 检查代码格式
4. **团队协作**: 确保团队成员都使用相同的格式化配置
