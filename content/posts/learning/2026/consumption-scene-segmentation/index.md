# 搜集数据

找到某一时间段中的订单数据以及下单用户的各个属性，包括订单总额、订单数量、用户年龄、性别、职业、收入等类目统计

![消费场景细分数据示意](/images/covers/study/CDP-1.png)

根据这张图，我们只能看出三种基础数据：

- 漏斗：Registered -> ATC -> Checkout -> Ordered
- 属性：年龄、性别、收入、职业、市场
- 来源与偏好：UTM、最近浏览类目/商品

产品化的思维就是需要我们根据这张图的基础数据进行分析：

**这些用户为什么会买家具？他们是在解决什么生活问题？**

# 划分“消费者/商品/场景”

按照“消费者/商品/场景”分析框架来对上图进行分析。

## 消费者

不能只看年龄性别，而是需要根据各种维度的数据找到家具决策强相关的用户群体

- 初次成家
- 已婚/有娃/二胎
- 一线城市白领
- 县城蓝领
- 居家办公
- ...

## 场景

对于家具消费者来说，用户购买的不只是简单的 Sofa/Bed/Table，而是：

- 搬新家时，快速的进行软装、家具配齐
- 小户型的家具会考虑节省空间
- 有孩子的父母会更加考虑耐用安全
- 有居家办公需求的会更加考虑功能性和审美
- 有招待客人需求的会考虑高端大气
- 户外布置
- ...

这就是家具电商更应该经营的“场景”。

## 商品/货物

不是孤立的、单一品类的商品，而是某一场景的解决方案

- “首次搬家”对应：床架 + 沙发 + 餐桌椅基础套装
- “小空间优化”对应：storage bed、extendable dining table、compact sofa
- “居家办公”对应：desk + chair + lighting + storage
- “有娃家庭”对应：耐污面料、圆角、易清洁、可收纳

# 使用结构化思维进行分析

结合上面的叙述，重新对上图进行分析，从单一的关注“卖了多少/谁加购了/谁消费了”。转向"什么场景的人在加购？哪个场景的人卡在了checkout？这个商品/套装解决了什么场景的问题"

所以下一步值得尝试的是使用这种结构花思维进行数据分析,

把现有 dashboard 从“类目/漏斗报表”改成“场景视图优先”。顶层先确定场景，再看人群、渠道、漏斗、商品。

1. 给每个场景定义。
   - 目标用户是谁
   - 典型触发时刻是什么
   - 需要哪些商品组合
   - 需要哪些内容支持决策
   - 最关键转化动作是什么
2. 用场景重做前台组织，而不是只靠类目树。
   - 例如首页、导航、PLP、推荐位、内容专题页都可以场景化。
3. 把营销、搜索、推荐也按场景组织。
   - 文章里专门批评了“拿后台类目直接做 C 端运营”的方式，这一点对家具电商很适用。

# 总结

不仅仅是家具行业，其他行业的产品也是如此，我们要知道用户需要解决什么问题，我们的商品是否提供了解决方案
