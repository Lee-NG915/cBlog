# 将需要解决的问题公式化

对于我这个产品初学者，看到的相关的数据报表只是一些简单的数字，注册用户数量、订单数量、加购数量、checkout率、payment率等，我无法关联出对应的产品逻辑。

学会从数据中分析出“消费场景”这个概念是很重要的，即用户为什么买？从哪里来？买的什么？

固定记住一个公式

```
场景 = 谁在买 × 为什么现在买 × 买什么组合 × 走什么路径
```

我们只需要从数据报表中找出这四个问题的答案

# 各个数据模块代表的真实含义

在这张图里每个模块，其实都对应场景框架的一部分：

User Profile Summary
作用：回答“谁在买”
例子：26-35、Female、高收入、Office/Remote Worker
注意：这只是人群特征，不是场景本身

Registered -> ATC -> Checkout -> Ordered
作用：回答“走到哪一步卡住了”
例子：加购多但 checkout 少，说明不是没兴趣，而是决策没过

Registered User First UTM
作用：回答“为什么来”
例子：facebook / cpc 更像灵感种草，affiliate 可能更接近比较决策

Last 7 Days Browsed Category/Product
作用：回答“他在解决什么家具问题”
例子：Beds + Storage Bed 很可能不是单纯看床，而是卧室收纳/新家布置

Registered User Behaviour Description
作用：回答“用户行为强度和价值”
例子：Session Count、Product Views、Cart Items Value、Email Opened

所以你分析时不要问：
**这个图里有什么结论？**
要问：
**这个模块能帮我回答场景定义里的哪一个问题？**

# 从报表数据梳理出“场景”

用一个很死板但有效的 4 步法。

## 1.先列 3 到 5 个“候选场景”

不要从数据开始，先从业务和商品开始想。对家具电商，最容易成立的通常是：

首次置家
卧室升级
客厅焕新
小空间收纳
居家办公
待客/餐厅布置
用报表去找“证据组合”

## 2.一个场景至少要有 4 类证据

不能只看一个维度：

入口证据：UTM / 渠道
兴趣证据：浏览类目 / 浏览商品
行为证据：ATC / Checkout / Order
人群证据：年龄 / 收入 / 职业 / 市场

## 3.把“证据组合”写成一句人话

比如不是写：
`Female 26-35, Beds, Facebook CPC`

而是写：
`26-35 岁女性，从 Facebook 被种草进入，近期集中看床和储物床，可能处于新家卧室布置或小空间收纳升级场景。`

## 4.再回头看这个场景值不值得经营

判断标准就 3 个：

- 人够不够多
- 转化值够不够高
- 问题够不够集中

如果三者都不明显，这个场景先不要做。

# 场景分析模版

| 模块              | 你要问的问题     | 产出       |
| ----------------- | ---------------- | ---------- |
| UTM / 渠道        | 用户为什么来     | 入口动机   |
| 浏览类目 / 商品   | 用户在看什么问题 | 需求主题   |
| 漏斗              | 用户卡在哪一步   | 决策阶段   |
| 画像              | 谁更像这类用户   | 人群轮廓   |
| 订单/AOV/行为强度 | 值不值得重点做   | 商业优先级 |

最后把它整理成一句完整场景定义：

**某类人，在某种触发下，为了解决某个生活问题，会浏览某些商品，并在某一步发生明显犹豫。这就是场景框架。**
