# Next.js ISR + Redis 共享缓存方案设计与实现

## 一、方案概述

### 1.1 问题背景

- 页面渲染耗时与后端压力现状
- 静态化与动态数据更新的平衡诉求

### 1.2 核心设计思路

- ISR 负责页面级静态化
- Redis 负责跨实例共享缓存
- Webhook 负责精准内容刷新

## 二、系统架构

### 2.1 组件拓扑

- Next.js 应用层
- Redis 缓存层（多实例高可用）
- CMS 与数据源（CMS系统 / 商品中台服务）

### 2.2 数据流向

- 请求命中缓存的完整链路
- 缓存未命中时的降级策略

## 三、缓存能力设计

### 3.1 数据缓存层

- 缓存源：CMS系统（内容管理）与商品中台服务（业务数据）
- 缓存 Key 设计与命名空间隔离
- TTL 与过期策略

### 3.2 页面缓存层

- ISR revalidate 周期配置
- 按路由细化的缓存规则

## 四、Redis 共享缓存实现

### 4.1 多实例一致性保障

- Redis Cluster 选型与分片策略
- 缓存穿透、击穿、雪崩的防护

### 4.2 序列化与存储优化

- 大数据对象的压缩策略
- 热点数据的本地二级缓存（可选）

## 五、Webhook 刷新机制

### 5.1 触发源与事件类型

- CMS 内容发布事件
- 商品中台数据变更事件

### 5.2 精准刷新策略

- 按页面路径定向清除
- 批量刷新与限流保护
