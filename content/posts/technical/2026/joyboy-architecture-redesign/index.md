# 企业级电商前端平台架构重构：从七年 legacy 到多端复用

## 一、背景与问题诊断

### 1.1 原前端应用的困境

- 消费者端与门店端代码高度耦合，维护成本极高
- 前端业务逻辑过度膨胀
- 系统已持续迭代 7 年，技术债务严重
- 门店业务逐步复用消费者端能力（O2O 场景）
- 多端 UI 组件与业务逻辑完全无法复用
- 测试覆盖率缺失，重构上线风险高
- 需求文档与设计文档双重缺失，沟通效率低
